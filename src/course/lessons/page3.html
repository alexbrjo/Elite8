Meet the Intel 8008

"eighty-oh-eight"

The Intel 8008 is an 8-bit microprocessor released in 1972 that powered early personal computers. One of these was the Datapoint 2200, which was a mass-produced programmable personal computer. It has 2 parts that you need to be familiar with: registers and flags and several others that are useful to know.

Registers

Now since you've just learned what registers are we can go over which are provided in the Intel 8008.

Scratch pad Registers

Seven 8-bit scratch pad registers which are for temporary value storage. These are most important to the programmer because these are similar to local variables in a higher-level language.

Flags

Most microprocessors today have a flag register, but the Intel 8008 does not. Flags are 1-bit values that are set as a result of an operation. They are similar to a returns value from a function. The Intel 8008 has 4 flags: sign, zero, parity and carry. Arithmetic operations will update all 4 of these flags when they are preformed.

Flag Condition
Sign set if the result of an operation is negative
Zero set if the result of an operation is zero
Parityset if the result of an operation is even
Carry only set if the result of an operation is greater than 255 or less than 0

Later you will learn that certain opcodes use these flags as conditionals and are very useful.

The 14-bit Address Bus

With 8 bits of data and an 8-bit address bus you can reach 256 (28) memory locations. Thats a very small amount of memory. Luckily the Intel 8008 has 14-bit address bus which can reach 16,384 (214) memory locations. Unfortunately a byte is only 8 bits, so two bytes of data is needed to reach all the possible memory locations in the scope of the bus. This is organized by having a high-order byte and a low order byte (the 2 most significant bits of the high-order byte are ignored). This is why the stack registers are 16-bit, so they can point to 14-bit memory locations.

Why learn about a chip from the 1970's?

Chips today are larger and more complicated. Some of the visuals in this course would be a lot larger and extremely difficult to emulate in JavaScript.

While you probably won't write assembly for the Intel 8008, you may write assembly for other other 8-bit microprocessors, which today still take up a large portion of the microprocessor market (40% in 2011). Additionally, the Intel 8008 was a key influencer of the x86 architecture.
